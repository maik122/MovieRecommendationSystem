#  Movie Recommendation System

A collaborative filtering recommendation engine built on the **MovieLens 100k** dataset, with a deployed [Streamlit](https://streamlit.io) web app. Pick three movies you love and get ten personalised recommendations — powered by three different algorithms.

**Live demo →** *(add your Streamlit Cloud URL here)*

---

## Overview

CineMatch implements and compares three recommendation approaches:

| Method | Friendly Name | How it works |
|---|---|---|
| Item-based CF | Similar Movies | Finds movies with the highest cosine similarity to your picks |
| User-based CF | Similar Viewers | Builds a pseudo-user profile, finds real users most similar to you, and surfaces what they rated highly |
| SVD | Personalised Pick | Matrix factorisation that decomposes the full ratings matrix into latent factors to predict your personal rating for every unseen film |

All three are evaluated with **Precision@K** and the SVD model is tuned via **GridSearchCV** across 81 hyperparameter combinations.

## How the App Works

1. **Pick three movies** you love from the full MovieLens catalogue
2. **Choose a recommendation style** — Similar Movies, Similar Viewers, or Personalised Pick
3. **Set how many results** you want (5–20)
4. Hit **Find My Movies** and get a ranked watchlist with match scores

## Model Training & Evaluation

### Algorithms

**Item-based Collaborative Filtering**
Computes cosine similarity between every pair of movies using the user-item rating matrix. Recommendations are scored by summing the similarity of each candidate movie to your three seed picks.

**User-based Collaborative Filtering**
Creates a synthetic user profile (rating your three picks as 5/5), computes cosine similarity against all 943 real users, then generates a weighted average rating for every unseen movie.

**SVD (Matrix Factorisation)**
Uses the [Surprise](https://surprise.readthedocs.io) library's SVD implementation. The ratings matrix is factorised into user and item latent factor matrices. GridSearchCV tunes four hyperparameters:

```python
param_grid = {
    'n_factors': [50, 100, 150],        # latent dimensions
    'n_epochs':  [20, 40, 60],          # training iterations
    'lr_all':    [0.002, 0.005, 0.01],  # learning rate
    'reg_all':   [0.02, 0.05, 0.1],     # regularisation
}
```

### Evaluation

| Metric | Description |
|---|---|
| Precision@K | Fraction of top-K recommendations that appear in the user's held-out test ratings (≥ 4 stars) |
| RMSE | Root mean squared error of SVD's predicted vs actual ratings on the test split |

An 80/20 train/test split is used throughout (`random_state=42`).

## Deployment (Streamlit Cloud)

> **Important:** `scikit-surprise` uses Cython extensions that are binary-incompatible across Python versions. The SVD model is therefore **not pickled** — instead, `svd_params.json` stores the best hyperparameters and the app retrains SVD from `u.data` at startup (cached via `@st.cache_resource`).

### Steps

1. Push your repo to GitHub — include `models/` and `ml-100k/` (see dataset note above)
2. Go to [share.streamlit.io](https://share.streamlit.io) → **New app** → connect your repo
3. Set **Main file path** to `app.py`
4. Deploy — first boot takes ~30 seconds while SVD retrains

### Why `packages.txt`?

`scikit-surprise` compiles C extensions during install and requires system build tools. `packages.txt` tells Streamlit Cloud to run `apt-get install build-essential` before the Python install step.

### Why `numpy<2.0`?

The pre-compiled Surprise `.pyx` binaries are linked against NumPy 1.x's C API. NumPy 2.0 introduced a breaking ABI change that causes an `ImportError` at runtime. Pinning `numpy<2.0` ensures compatibility.

## Dataset

**MovieLens 100k** — collected by the GroupLens Research Project, Department of Computer Science and Engineering, University of Minnesota.

| Stat | Value |
|---|---|
| Ratings | 100,000 |
| Users | 943 (each rated at least 20 movies) |
| Movies | 1,682 |
| Rating scale | 1–5 stars |
| Collection period | Sep 19, 1997 – Apr 22, 1998 |
| User demographics | Age, gender, occupation, zip code |
| Movie metadata | Title, release date, IMDb URL, 19 genre flags |

### Key files used

| File | Contents |
|---|---|
| `u.data` | Tab-separated: `user_id`, `movie_id`, `rating`, `timestamp` |
| `u.item` | Pipe-separated: movie id, title, release date, IMDb URL, 19 binary genre columns |
| `u.genre` | List of the 19 genre labels |
| `u.user` | Tab-separated: `user_id`, `age`, `gender`, `occupation`, `zip_code` |
